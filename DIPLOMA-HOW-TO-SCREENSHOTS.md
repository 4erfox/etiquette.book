# Практическое руководство: КАК СОЗДАВАТЬ И СКРИНИТЬ ОШИБКИ ДЛЯ ДИПЛОМА

**Назначение:** Это руководство поможет вам умышленно вызвать различные ошибки, заскринить их и потом исправить для демонстрации в дипломном отчёте.

---

## 🎬 ПРОЦЕСС СКРИНШОТИРОВАНИЯ ОШИБОК

### Инструменты для скриншотов:
- **Встроенный ScreenShot Windows:** `Win + Shift + S`
- **VS Code:** Встроенный скриншот терминала
- **Более продвинуто:** Obsidian, Greenshot, ShareX (для аннотаций)

## 1️⃣ ОШИБКА: НЕСОВМЕСТИМАЯ ВЕРСИЯ NODE.JS

### Как создать ошибку:

**Условие:** У вас должен быть Node.js версии < 18

**Вариант 1: Если у вас Node < 18 (повезло!)**
```bash
node --version
# v16.14.2
```

Просто выполнить:
```bash
npm install
```

Получится ошибка в терминале ✅

**Вариант 2: Если у вас Node >= 18 (эмулировать ошибку)**

```bash
# Отредактировать package.json
"engines": {
  "node": ">=22.0.0"  # Версия выше текущей!
}

npm install
# Должна появиться ошибка ✅
```

### Как заскринить:

1. **PowerShell / Terminal:**
   ```
   Наведите мышь на терминал → Win + Shift + S → выделите область с ошибкой
   Сохраните в DIPLOMA-SCREENSHOTS/01_initialization/error-node-version.png
   ```

2. **Получится скрин примерно такой:**
   ```
   ┌─────────────────────────────────────────────┐
   │ > npm install                               │
   │                                             │
   │ npm ERR! The engines in this package        │
   │ require node >=22.0.0                       │
   │ npm ERR! You are currently using node 20.x  │
   │                                             │
   │ npm ERR! A complete log of this run...      │
   └─────────────────────────────────────────────┘
   ```

### Как исправить:

```bash
# Вернуть package.json обратно или обновить Node
"engines": {
  "node": ">=18.0.0"
}

npm install
# Теперь работает ✅
```

### Скрин после исправления:

```
┌────────────────────────────────────────────────┐
│ > npm install                                  │
│                                                │
│ added 245 packages, and audited 310 packages   │
│ in 45s                                         │
│                                                │
│ 2 packages have known vulnerabilities          │
│ run `npm audit fix` to fix them, or           │
│ `npm audit` for details                        │
└────────────────────────────────────────────────┘
```

## 2️⃣ ОШИБКА: JWT_SECRET НЕ УСТАНОВЛЕНА

### Как создать ошибку:

**Шаг 1:** Отредактировать `admin.env`

```env
# admin.env — сделать пустой или неверный
JWT_SECRET=        # ← ПУСТО!
# или
JWT_SECRET=ЗАМЕНИ_МЕНЯ   # ← Как в шаблоне
```

**Шаг 2:** Запустить сервер

```bash
npm run server
```

**Результат:**
```
❌  Задай JWT_SECRET в переменных окружения Render!

(node:8392) ExitCode: 1
```

### Скрин ошибки:

Скринить весь вывод терминала от `npm run server` до ошибки.

```

│ > etiquette-book@1.0.0 server              │

│     окружения Render!                       │
│                                             │
│ PS E:\Проект123\админ панель вместе>       │
└─────────────────────────────────────────────┘
```

### Как исправить:

**Шаг 1:** Сгенерировать ключ

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

**Скрин генерации:**

**Шаг 2:** Скопировать в `admin.env`

**Шаг 3:** Запустить снова

## 3️⃣ ОШИБКА: ПОРТ УЖЕ ЗАНЯТ

### Как создать ошибку:

**Вариант 1: Двойной запуск (простейший способ)**

```bash
# Терминал 1:
npm run server

# Терминал 2 (пока первый запущен):
npm run server
```

**Вариант 2: Запустить что-то на порту 7778**

```bash
# Запустить простой HTTP сервер на 7778:
npx http-server -p 7778
```

**Результат ошибки:**
```
Error: listen EADDRINUSE: address already in use :::7778
    at Server.setupListenHandle [as _listen2]
```

### Скрин ошибки:

```
┌──────────────────────────────────────────┐
│ PS C:\> npm run server                   │
│                                          │
│ Error: listen EADDRINUSE: address        │
│ already in use :::7778                   │
│     at Server.setupListenHandle          │
│     at net.js:1058:13                    │
│                                          │
│ PS C:\>                                  │
└──────────────────────────────────────────┘
```

### Как исправить:

**Вариант A: Завершить процесс на порту**

**Windows PowerShell:**
```powershell
netstat -ano | findstr :7778
taskkill /PID 5234 /F
```

**Скрины:**
1. Скринить команду `netstat` с выводом
2. Скринить команду `taskkill` 
3. Скринить повторный запуск `npm run server` — работает ✅

**Вариант B: Использовать другой порт**

```env
# admin.env
PORT=7779
```

Запустить:
```bash
npm run server
```

**Скрин успеха:**
```
┌──────────────────────────────────────────┐
│ PS > npm run server                      │
│                                          │
│ ✅ Server running at http://0.0.0.0:7779│
└──────────────────────────────────────────┘
```

## 5️⃣ ОШИБКА: CORS БЛОКИРУЕТ ЗАПРОС

### Как создать ошибку:

**Шаг 1:** Закомментировать CORS в `server.js`

```javascript
// Найти:
// app.use(cors({...}));

// Заменить на:
// app.use(cors({...}));  ← КОММЕНТАРИЙ

// Или удалить строку вообще
```

**Шаг 2:** Перезапустить сервер

```bash
npm run server
```

**Шаг 3:** В браузере отправить запрос

```javascript
// В браузере консоль (F12):
fetch('http://localhost:7778/api/pages')
  .then(r => r.json())
  .then(d => console.log(d))
```

### Скрин ошибки (браузер F12):

```
┌────────────────────────────────────────────┐
│ GET http://localhost:7778/api/pages        │
│                                            │
│ CORS error:                                │
│ Access to XMLHttpRequest at               │
│ 'http://localhost:7778/api/pages'         │
│ from origin 'http://localhost:5173'       │
│ has been blocked by CORS policy:          │
│                                            │
│ No 'Access-Control-Allow-Origin' header   │
│ is present on the requested resource.     │
└────────────────────────────────────────────┘
```

**Или в таблице Network (F12):**

| Name | Status | Type |
|------|--------|------|
| pages | (blocked) | xhr |

### Как исправить:

**Шаг 1:** Раскомментировать CORS в `server.js`

```javascript
// Найти:
// app.use(cors({...}));

// Раскомментировать:
app.use(cors({
  origin: 'http://localhost:5173',
  credentials: true
}));
```

**Шаг 2:** Перезапустить сервер

```bash
npm run server
```

**Шаг 3:** Повторить запрос в браузере

```javascript
fetch('http://localhost:7778/api/pages')
  .then(r => r.json())
  .then(d => console.log(d))
```

### Скрин успеха (браузер консоль):

```
┌────────────────────────────────────────────┐
│ GET http://localhost:7778/api/pages 200   │
│                                            │
│ Response: {pages: Array(0)}                │
│           pages: []                        │
└────────────────────────────────────────────┘
```

```javascript
// public/scripts/index.js
```

```bash
npm run build
```

## 📸 СОВЕТЫ ПО СКРИНШОТИРОВАНИЮ

### Лучшие практики:

1. **Полный контекст:**
   - Скринить весь терминал (не только ошибку)
   - Включать команду, которую вы запустили
   - Включать путь проекта (E:\Проект123\...)

2. **Хороший скрин:**
   - Читаемый размер шрифта
   - Видна вся ошибка
   - Видны все детали (строки, файлы)

3. **Плохой скрин:**
   - Слишком мелкий текст (не видно)
   - Обрезанная ошибка (непонятно)
   - Лишние элементы (отвлекают)

### Инструменты для аннотаций:

После скриншота добавить стрелки/текст:
- **Paint 3D** (встроен в Windows)
- **Greenshot** (бесплатно)
- **ShareX** (мощнее)

Примеры аннотаций:
```
┌─────────────────────────────┐
│ ❌ ОШИБКА               ↑   │
│                         │   │
│ Error: Cannot find      │   │
│ module 'express'        │   │
│                         │   │
│ ← Здесь указана      ← Файл │
│   причина                   │
└─────────────────────────────┘
```
