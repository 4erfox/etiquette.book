# 🎓 ПОЛНОЕ РУКОВОДСТВО ПО ОШИБКАМ ДЛЯ ДИПЛОМА

**Быстрый старт: как использовать эти документы для создания отчёта**

---

## 📂 ЧТО НАХОДИТСЯ В ЭТОЙ ПАПКЕ?

### 1. **DIPLOMA-ERRORS-GUIDE.md** ← ГЛАВНЫЙ ДОКУМЕНТ
**Содержит:** Полное описание всех типичных ошибок с примерами

**Когда читать:** Если нужна полная информация об ошибке

**Структура:**
- 8 частей (инициализация, конфигурация, зависимости, сервер, фронтенд, API, build, деплой)
- 20+ типичных ошибок с детальным разбором
- Для каждой: скрин ошибки, код исправления, скрин успеха
- Краткая шпаргалка в конце

### 2. **DIPLOMA-QUICK-COMMANDS.md** ← ДЛЯ НЕТЕРПЕЛИВЫХ
**Содержит:** Только команды для быстрого воспроизведения ошибок

**Когда читать:** Если нужна быстрая справка для создания ошибок

**Пример:**
```bash
# Заскринить ошибку CORS
# Отредактировать server.js (закомментировать cors)
npm run server

# ❌ Ошибка готова для скриншота
```

### 3. **DIPLOMA-HOW-TO-SCREENSHOTS.md** ← ПРАКТИЧЕСКОЕ РУКОВОДСТВО
**Содержит:** Пошаговые инструкции по созданию и скриншотированию каждой ошибки

**Когда читать:** Если не знаете, как физически создать ошибку

**Пример:**
```
Ошибка 1.1 – Несовместимая версия Node.js
┌─────────────────────────────────────────┐
│ Как создать:                            │
│ 1. Отредактировать package.json         │
│ 2. npm install                          │
│ 3. Заскринить результат                 │
│                                         │
│ Как исправить:                          │
│ 1. Обновить Node.js                     │
│ 2. npm install                          │
│ 3. Заскринить успех                     │
└─────────────────────────────────────────┘
```

### 4. **DIPLOMA-FORMATTING-TEMPLATE.md** ← ОФОРМЛЕНИЕ ОТЧЁТА
**Содержит:** Готовые примеры оформления ошибок для дипломной работы

**Когда читать:** Если нужно красиво оформить ошибки в итоговом отчёте

**Включает:**
- Структурированные примеры (4 полных примера)
- Таблицы с анализом
- Рекомендации по безопасности
- Советы по написанию

## 🚀 БЫСТРЫЙ СТАРТ: 3 ШАГА

### Шаг 1️⃣: Прочитайте краткую инструкцию
```
Откройте: DIPLOMA-HOW-TO-SCREENSHOTS.md
Прочитайте раздел: "ИНСТРУЦИЯ: КАК ИСПОЛЬЗОВАТЬ"
Время: ~5 минут
```

### Шаг 2️⃣: Выполните команды для создания ошибок
```
Откройте: DIPLOMA-QUICK-COMMANDS.md
Скопируйте команду нужной ошибки
Выполните в терминале
Заскринить результат (Win + Shift + S)
Повторите для 8-12 разных ошибок
Время: ~2-3 часа
```

### Шаг 3️⃣: Оформите ошибки в отчёт
```
Откройте: DIPLOMA-FORMATTING-TEMPLATE.md
Используйте структуру из примеров
Добавьте свои скриншоты
Напишите описание и анализ
Время: ~1-2 часа
```

## 📋 РЕКОМЕНДУЕМЫЙ ПОРЯДОК ОШИБОК

Для полного отчёта выберите 8-15 ошибок в этом порядке:

### 🟢 Основные (обязательные):
1. ✅ **Версия Node.js < 18** — классическая проблема новичков
2. ✅ **JWT_SECRET не установлена** — security issue
3. ✅ **Пароль администратора неверен** — authentication
4. ✅ **Порт уже занят** — network issue
5. ✅ **CORS блокирует запрос** — API issue

### 🟡 Дополнительные:
6. ✅ Модуль express не найден
7. ✅ Build не работает
8. ✅ Синтаксическая ошибка в коде

### 🔵 Продвинутые (если есть время):
9. ✅ Деплой на Render.com
10. ✅ Проблема с памятью при установке
11. ✅ WebSocket HMR не работает
12. ✅ Повреждённый package-lock.json

## 🎬 ИНСТРУКЦИЯ ПО СКРИНШОТИРОВАНИЮ

### Основные инструменты:
- **Windows встроенный:** Win + Shift + S
- **VS Code:** Ctrl + Shift + P → "Screenshot"
- **Advanced:** Greenshot (для аннотаций)

### Структура для сохранения скриншотов:
```
DIPLOMA-SCREENSHOTS/
├── 01-node-version-error.png
├── 01-node-version-fixed.png
├── 02-jwt-secret-error.png
├── 02-jwt-secret-fixed.png
├── 03-port-busy-error.png
├── 03-port-busy-fixed.png
└── ... (и т.д.)
```

### Что скринить:
✅ **Полный скрин терминала** (видна команда и вывод)
✅ **Видно имя пользователя и путь проекта**
✅ **Вся ошибка целиком** (не обрезанная)
✅ **Скрины ПОСЛЕ исправления** (для сравнения)

## 🔐 ВАЖНЫЕ ЗАМЕЧАНИЯ ПО БЕЗОПАСНОСТИ

⚠️ **НИКОГДА НЕ ПОКАЗЫВАЙТЕ:**
- Реальные пароли администратора
- Реальные JWT_SECRET ключи
- Реальные credentials/tokens
- Personal API keys

✅ **ИСПОЛЬЗУЙТЕ ВМЕСТО ЭТОГО:**
- Примеры вида: `$2b$10$XyZ1a2b3c4d5e6f7g8h9...` (обрезанный хэш)
- Демо ключи: `demo-jwt-key-12345` (явно обозначенные как демо)
- Объяснение: "В production используются реальные ключи из переменных окружения"

## 📞 ЕСЛИ ЧТО-ТО НЕ РАБОТАЕТ

### Ошибка воспроизводится не так, как описано?
→ Проверьте версию Node.js, npm и пакетов в DIPLOMA-ERRORS-GUIDE.md

### Скриншот не получается?
→ Используйте встроенное Win + Shift + S или попробуйте Alt + Print Screen

### Не знаете, как исправить?
→ Используйте DIPLOMA-QUICK-COMMANDS.md с готовыми решениями

### Нужно красиво оформить?
→ Скопируйте структуру из DIPLOMA-FORMATTING-TEMPLATE.md
